# Security Policy

## Supported Versions

This project is in active development and does not provide formal support guarantees yet.  
Security issues are still taken seriously and will be addressed on a best-effort basis.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Preview Environment Platform:

1. **Do not** open a public GitHub issue.
2. Send a private report to: `mikesydocompan@gmail.com`.
3. Include:
   - a clear description of the issue,
   - steps to reproduce,
   - expected and actual behavior,
   - any relevant logs or configuration details (redact secrets).

You can also use GitHub’s private advisory feature if enabled for this repository.

## Scope

This project is intended as a template and learning resource.  
When using it in production, you are responsible for:

- securing your Kubernetes cluster,
- managing cloud credentials and secrets safely,
- reviewing network exposure and ingress configuration.
